## Hard separation model

This repo now keeps source pages in separate folders. `src/pages/` is generated and ignored by git.

| Folder | Purpose |
|---|---|
| `src/professional/` | GitHub Pages only: professional homepage routes, projects, technical blog, UHI, Glass Fidelity. |
| `src/realms/` | Cloudflare only: Kosh, astrology, cooking, essays, reading notes, dharma/manifesto, ancient-sciences placeholders. |
| `src/shared-pages/` | Shared generated pages such as `robots.txt.ts`, `sitemap.xml.ts`, `404.astro`. |
| `src/components/ProfessionalHome.astro` | GitHub Pages homepage component. |
| `src/components/RealmsHome.astro` | Cloudflare Realms homepage component. |

`src/pages/` is generated by:

```bash
node scripts/prepare-pages.mjs
```

The generator copies only the selected target into `src/pages/`, then writes the correct homepage wrapper.

## Build scripts

```json
"build:github": "pnpm prepare:professional && SITE_MODE=professional astro build",
"build:cloudflare": "pnpm prepare:realms && SITE_MODE=realms astro build && rm -rf dist/blog/uhi-bengaluru dist/blog/glass-fidelity dist/projects/terrain-safety dist/projects/python-api"
```

The Cloudflare build also deletes any professional artifacts from `dist/` as a safety belt.

## Maintenance rules

1. Add professional-only pages under `src/professional/`.
2. Add Realms-only pages under `src/realms/`.
3. Do **not** add real source files directly under `src/pages/`; it is generated and ignored.
4. If a page must be shared, place it under `src/shared-pages/` and make it `SITE_MODE` aware if needed.
5. Always test both builds before deploying:

```bash
pnpm build:github
pnpm build:cloudflare
```

6. To prove UHI is not on Cloudflare:

```bash
test ! -e dist/blog/uhi-bengaluru && echo "Cloudflare output has no UHI"
```

7. Deploy Cloudflare manually if needed:

```bash
pnpm deploy:cloudflare
```

8. GitHub Pages deploys automatically on push to `master` via `.github/workflows/deploy.yml`.
